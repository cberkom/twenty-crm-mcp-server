# Version 0.6.0 - Power Features Implementation Plan

**Status:** Ready to Implement
**Created:** November 14, 2025
**Target Completion:** Q2 2025
**Estimated Effort:** 10-14 hours

---

## Overview

Version 0.6.0 adds two major features to the Twenty CRM MCP Server:
1. **Attachments & Files** - Document management
2. **Workflow Automation** - Process automation

This brings the tool count from 34 to 46-50 tools.

## Phase 1: Attachments Domain (Priority 1)

**Effort:** 4-6 hours
**Tools:** 4-6
**Status:** 🟡 Not Started

### API Verification ✅

Confirmed available in `scenerii_twenty_rest_response.json`:
- `/attachments` - GET, POST
- `/attachments/{id}` - GET, PATCH, DELETE
- `/batch/attachments` - POST, PUT, DELETE

### Schema Analysis

**Attachment Object:**
```typescript
interface Attachment {
  id: string;
  name: string;
  fullPath: string;
  type?: string;  // Deprecated
  fileCategory?: "ARCHIVE" | "AUDIO" | "IMAGE" | "PRESENTATION" |
                  "SPREADSHEET" | "TEXT_DOCUMENT" | "VIDEO" | "OTHER";

  // Relationships
  taskId?: string;
  opportunityId?: string;
  companyId?: string;
  personId?: string;
  authorId?: string;
  workflowId?: string;
  dashboardId?: string;

  // Metadata
  createdBy?: {
    source: "EMAIL" | "CALENDAR" | "WORKFLOW" | "AGENT" |
            "API" | "IMPORT" | "MANUAL" | "SYSTEM" | "WEBHOOK";
  };
  createdAt: string;
  updatedAt?: string;
  deletedAt?: string | null;
}
```

### Tools to Implement

#### 1. `upload_attachment` / `create_attachment`
**Purpose:** Upload a file and attach to a record
**Input:**
```typescript
interface UploadAttachmentInput {
  name: string;
  fullPath: string;  // File path or URL
  fileCategory?: AttachmentCategory;

  // Link to one of:
  taskId?: string;
  opportunityId?: string;
  companyId?: string;
  personId?: string;
  workflowId?: string;
  dashboardId?: string;
}
```

**GraphQL Mutation:**
```graphql
mutation CreateAttachment($input: AttachmentCreateInput!) {
  createAttachment(data: $input) {
    id
    name
    fullPath
    fileCategory
    taskId
    opportunityId
    companyId
    personId
    createdAt
    updatedAt
  }
}
```

**Challenges:**
- File upload mechanism (need to understand Twenty's file handling)
- Base64 encoding vs direct file path
- Large file support
- MIME type detection

---

#### 2. `get_attachment`
**Purpose:** Retrieve attachment details by ID
**Input:** `{ id: string }`

**GraphQL Query:**
```graphql
query GetAttachment($id: ID!) {
  attachment(id: $id) {
    id
    name
    fullPath
    fileCategory
    type
    taskId
    opportunityId
    companyId
    personId
    authorId
    createdBy {
      source
    }
    createdAt
    updatedAt
  }
}
```

---

#### 3. `list_attachments`
**Purpose:** List attachments with filtering
**Input:**
```typescript
interface ListAttachmentsParams {
  taskId?: string;
  opportunityId?: string;
  companyId?: string;
  personId?: string;
  fileCategory?: AttachmentCategory;
  searchTerm?: string;
  limit?: number;
}
```

**GraphQL Query:**
```graphql
query ListAttachments($filter: AttachmentFilterInput, $limit: Int) {
  attachments(filter: $filter, limit: $limit) {
    edges {
      node {
        id
        name
        fullPath
        fileCategory
        taskId
        opportunityId
        companyId
        personId
        createdAt
      }
    }
    pageInfo {
      hasNextPage
      endCursor
    }
  }
}
```

---

#### 4. `delete_attachment`
**Purpose:** Remove an attachment
**Input:** `{ id: string }`

**GraphQL Mutation:**
```graphql
mutation DeleteAttachment($id: ID!) {
  deleteAttachment(id: $id) {
    id
  }
}
```

### Technical Challenges & Solutions

**Challenge 1: File Upload Mechanism**
- Need to research Twenty's file upload API
- May need to handle multipart/form-data
- Consider base64 encoding for binary data
- Solution: Start with fullPath (URL or path), research upload API

**Challenge 2: File Categories**
- 8 different categories to support
- Need MIME type detection or user specification
- Solution: Make fileCategory optional, let user specify

**Challenge 3: Multiple Relationship Types**
- Attachment can link to 7 different entity types
- Only one should be set at a time
- Solution: Accept all as optional, validate at least one is provided

### Schema Analysis

**Workflow Object:**
```typescript
interface Workflow {
  id: string;
  name: string;
  statuses?: string[];  // Workflow stages/statuses
  position?: number;

  // Relationships
  workflowVersionsCollection?: WorkflowVersion[];
  workflowRunsCollection?: WorkflowRun[];

  // Metadata
  createdAt: string;
  updatedAt?: string;
  deletedAt?: string | null;
}

interface WorkflowVersion {
  id: string;
  name: string;
  trigger?: object;  // Trigger configuration
  status?: string;
  workflowId: string;
  createdAt: string;
  updatedAt?: string;
}

interface WorkflowRun {
  id: string;
  name: string;
  position?: number;
  workflowId: string;
  workflowVersionId?: string;
  startedAt?: string;
  endedAt?: string;
  status?: string;
  output?: object;
  createdAt: string;
  updatedAt?: string;
}

interface WorkflowAutomatedTrigger {
  id: string;
  name: string;
  settings?: object;
  workflowVersionId: string;
  createdAt: string;
  updatedAt?: string;
}
```

### Tools to Implement

#### Core Workflow Management

1. **`create_workflow`**
   - Create new automation workflow
   - Input: name, statuses, position

2. **`get_workflow`**
   - Get workflow by ID
   - Include versions and runs

3. **`list_workflows`**
   - List all workflows with filtering
   - Support search and pagination

4. **`update_workflow`**
   - Update workflow configuration
   - Modify statuses, name, position

5. **`delete_workflow`**
   - Remove workflow (soft delete)

#### Workflow Versions

6. **`list_workflow_versions`**
   - List versions for a workflow
   - Show version history

7. **`get_workflow_version`**
   - Get specific version details
   - Include trigger configuration

#### Workflow Execution

8. **`list_workflow_runs`**
   - View execution history
   - Filter by workflow, status, date range

9. **`get_workflow_run`**
   - Get run details
   - Show output, duration, status

#### Optional Advanced Tools

10. **`list_workflow_triggers`**
    - List automated triggers
    - Show trigger settings

### Technical Challenges & Solutions

**Challenge 1: Complex Nested Structure**
- Workflows contain versions and runs
- Need to handle nested queries efficiently
- Solution: Use GraphQL fragments and proper pagination

**Challenge 2: Trigger Configuration**
- Triggers have flexible settings object
- May need schema validation
- Solution: Accept generic object, document common patterns

**Challenge 3: Execution Tracking**
- Run status management
- Output handling
- Solution: Proper status enum, flexible output object

**Challenge 4: Version Control**
- Managing workflow versions
- Active version tracking
- Solution: Clear version status field, version history queries
